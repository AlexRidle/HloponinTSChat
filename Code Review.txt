Code Review [RUS]

1) Stack Trace выводится пользователю, а не логируется
2) Некоторые исключения не логируют stack trace, либо вовсе отсутствует логирование.
3) Неправильное наименование классов, методов и переменных.
4) Классы не разбиты по пакетам.
5) Такие переменные как порт и хост должны находиться либо в отдельном файле с настройками либо в классе с константами.
6) Местами встречается неформатированный код. В IntelliJ комбинация ctrl + alt + l зачастую решает эту проблему автоматически.
7) Создаются переменные без которых можно обойтись, либо вовсе нигде не используются.
8) Имеются классы с идентичным поведением и переменными, практически ничем не отличающиеся. Необходимо использовать наследование (Например унаследовать клиента и агента от класса "Пользователь").
9) Встречается дубликат кода, который можно избежать, более детально проработав метод.
10) Уровень доступа у некторых методов public, когда его можно сделать private.
11) Необходимо импортировать только те классы, которые используются в коде. Не нужно импортировать сразу весь пакет.
12) При логировании местами выбран неправильный тип. (Например ошибка логируется как INFO, а не как ERROR)
13) Отсутствуют тесты для приложения.
